# 博客维护指南

> 个人思考博客（CWR&L）的结构说明与维护手册。改任何东西前，先在这里找对应位置。

---

## 一、技术栈

| 层 | 用什么 | 说明 |
|---|---|---|
| 框架 | Next.js 16（App Router + Turbopack） | 页面 = `src/app/` 下的文件夹 |
| 语言 | TypeScript + React 19 | |
| 样式 | Tailwind CSS v4 + `@tailwindcss/typography` | 颜色/间距写在 class 里；主题色集中在 `globals.css` |
| 内容 | MDX 文件 + gray-matter（读 frontmatter）+ next-mdx-remote（渲染） | 文章是 `content/` 下的 `.mdx` |
| MDX 增强 | remark-gfm（表格等）+ rehype-slug（标题锚点，目录跳转用） | |
| 可视化编辑 | Keystatic（本地模式） | 配置在 `keystatic.config.ts` |
| 动画 | framer-motion | Hero、卡片、搜索弹窗、花瓣 |
| 图标 | lucide-react | |
| 字体 | Inter（next/font） | |
| 部署 | Vercel + pnpm | push 到 `main` 自动构建上线 |
| 包管理 | **pnpm**（不是 npm！） | 装依赖用 `pnpm add`，别用 `npm install` |

## 二、目录结构

```
content/                  ← 你的文章(.mdx)，按板块分目录
  blog/  notes/  reading/  life/
public/images/            ← Keystatic 上传的图片(按板块自动归类)
src/
  app/                    ← 路由(每个文件夹 = 一个网址)
    page.tsx              ← 首页
    blog/   page.tsx + [slug]/page.tsx     ← 文章:列表 + 详情
    notes/  page.tsx + [slug]/page.tsx     ← 随笔
    reading/page.tsx + [slug]/page.tsx     ← 读书
    life/   page.tsx + [slug]/page.tsx     ← 生活
    tags/   page.tsx + [tag]/page.tsx      ← 标签聚合
    about/page.tsx        ← 关于
    keystatic/ + api/keystatic/            ← 可视化后台(仅本地可用)
    layout.tsx            ← 全站外壳(<html>、字体、站点元信息、主题防闪脚本)
    globals.css           ← ⭐ 所有颜色 / 主题 / 文章排版样式
    proxy.ts              ← 生产环境屏蔽 /keystatic(返回 404)
  components/             ← 所有 UI 组件
  lib/
    content.ts            ← ⭐ 统一内容加载器(板块定义、草稿/置顶、标签聚合、相邻文章)
    posts.ts notes.ts reading.ts life.ts   ← 各板块薄封装
keystatic.config.ts       ← 后台字段配置(4 个板块的栏目与字段)
scripts/
  content-autosync.mjs    ← 监听 content/ 与 public/images/，自动提交推送
  save.mjs                ← 一键 add+commit+push(含代码)
```

### 组件清单（`src/components/`）
- **导航/外壳**：`navbar`（顶部胶囊导航 + 主题切换 + 汉堡菜单）、`site-chrome`（鼠标光效/搜索/侧栏等装饰层的总开关）、`right-sidebar`（大屏右侧 NOW/回顶）、`back-to-top`、`mouse-glow`、`scroll-progress`
- **首页**：`hero-content`（左侧文案 slogan）、`hero-visual`（右侧暗色星球/亮色花束）、`top-cards`（四板块卡片）、`content-section`（近期文章 + 近期读书）、`learning-log`（随笔轮播）、`timeline-section`（金句 + 页脚）
- **文章**：`table-of-contents`（目录）、`post-nav`（上/下篇）、`tag-link`（可点击标签）、`pin-badge`（置顶徽章）、`draft-badge`（草稿标记）、`pre-copy`（代码块复制）
- **搜索**：`search-modal`（⌘K 全站搜索）
- **导航动作**：`nav-actions`（标签 + 写文章 + 编辑此篇 三合一；标签公开，写文章/编辑仅本地 dev；电脑端在右侧栏、移动端在汉堡菜单，三者用不同色调区分）、`keystatic-bar`（后台「回到首页/查看文章」，仅 dev）

## 三、主题与配色系统（调色前必看）

- `<html data-theme="dark|light">`，值存浏览器 localStorage（键 `blog-theme`），导航栏右侧圆点切换。
- 防闪烁：`layout.tsx` 里有段内联脚本，JS 加载前先把主题写到 `<html>`。
- **所有颜色都在 `src/app/globals.css`**：
  - 默认规则（无 `data-theme`）= **暗色主题**
  - `[data-theme="light"] xxx` = **亮色主题**覆盖
- **强调主色**在文件顶部：
  - 暗色：`:root { --accent-rgb: 168, 85, 247 }`（紫）
  - 亮色：`[data-theme="light"] { --accent-rgb: 82, 166, 108 }`（森林绿）
  - 换主色改这两个值即可。

### 「我想改 X，去哪改」速查表

| 想改的东西 | 去哪改 |
|---|---|
| 强调主色（紫/绿） | `globals.css` 顶部 `:root` 与 `[data-theme="light"]` 的 `--accent-rgb` |
| 亮色主题任意颜色 | `globals.css` 里所有 `[data-theme="light"] ...` 段落 |
| 亮色文章正文字色 | `globals.css` 中 `[data-theme="light"] .prose-invert p/li/h2/strong/a...`（深绿系那段 `!important`） |
| 暗色文章正文字色 | 4 个详情页里的 `prose-p:text-zinc-400`、`prose-headings:text-white` 等 class |
| 全站暗色背景 / 字色 | `globals.css` 的 `body { background; color }` |
| 导航菜单项 | `src/components/navbar.tsx` 的 `NAV_ITEMS` |
| 主题切换的圆点颜色/名称 | `navbar.tsx` 的 `THEMES` |
| 首页文案 / slogan | `src/components/hero-content.tsx`（标题、副标、徽章） |
| 首页板块与顺序 | `src/app/page.tsx` + 对应组件 |
| 页脚 | `src/components/timeline-section.tsx` |
| 后台栏目/字段 | `keystatic.config.ts` |
| 列表每页篇数 | 4 个列表页里的 `<Paginated perPage={8} ...>`（改数字即可） |

> 改完保存，本地 `pnpm dev` 会热更新，立即可见。

## 四、内容创作工作流（日常）

1. 终端跑 `pnpm write`（同时起本地服务 + 内容自动同步）
2. 浏览器打开 `http://localhost:3000/keystatic`（或点任意页面左下角绿色「写文章」）
3. 在后台可视化写：标题、URL（英文短横线，作文件名与链接）、日期、摘要、标签、封面、正文
4. **保存** → 约 6 秒后自动 `commit + push` → Vercel 自动重建上线（全程不碰 git）
5. 写完按一次 `Ctrl+C` 停止

### 字段技巧
- **草稿**：勾「草稿」→ 该篇线上隐藏（列表搜不到、直链 404），**本地仍可见且带「草稿」徽章**，方便预览；想发布就取消勾选。
- **置顶**：勾「置顶」→ 自动排到列表顶部，带「置顶」徽章。
- **标签**：跨板块串联同主题；详情页标签可点 → `/tags/<标签>`；`/tags` 是总览（页脚有入口）。
- **图片**：正文直接拖入/粘贴、封面点上传——自动存到 `public/images/<板块>/`，路径自动处理（autosync 会一并提交，不会裂图）。

### 改了代码（不是内容）怎么同步
- 一键：`pnpm save`（= `git add -A` + commit + push，自动生成提交信息）
- 带说明：`pnpm save "改了导航栏配色"`

## 五、自动化脚本

| 命令 | 作用 |
|---|---|
| `pnpm dev` | 只起本地开发服务器 |
| `pnpm write` | 起 dev + 内容自动同步（写文章时用这个） |
| `pnpm autosync` | 只起内容自动同步（监听 `content/` 与 `public/images/`，防抖后自动提交推送） |
| `pnpm save [信息]` | 一键提交并推送所有改动（含代码） |
| `pnpm build` | 本地生产构建（排错、上线前验证用） |

环境变量（可选）：`AUTOSYNC_DEBOUNCE=10000`（防抖毫秒）、`AUTOSYNC_BRANCH`、`SAVE_BRANCH`。

> 为什么内容自动、代码手动：写一半的代码若自动推送，Vercel 构建会失败。所以代码用 `pnpm save` 由你决定时机。

## 六、UI / 功能升级

### 小改（颜色 / 文案 / 间距）
照「速查表」改 `globals.css` 或对应组件即可。

### 新增一个板块（例如「影评」）—— 固定 6 步
1. 建目录 `content/movies/`
2. `src/lib/` 加封装 `movies.ts`（仿 `reading.ts`）
3. `src/lib/content.ts` 的 `SECTIONS` 加一项（加了之后该板块**自动进搜索和标签系统**）
4. `keystatic.config.ts` 加一个 collection（仿现有，含 draft/pinned）
5. 建页面 `src/app/movies/page.tsx`（列表）和 `src/app/movies/[slug]/page.tsx`（详情），仿 `reading`
6. `src/components/navbar.tsx` 的 `NAV_ITEMS` 加导航项

### 值得加的功能（按性价比）
- **每篇独立 SEO + 分享卡片**：用 `generateMetadata` + Open Graph 图，分享到微信/X 预览更专业
- **RSS 订阅**（`/rss.xml`）：思考类读者常用
- **sitemap / robots**：利于搜索引擎收录
- **评论**：Giscus（基于 GitHub Discussions，免费、零后端，契合 git 工作流）
- **阅读时长 / 字数统计**

## 七、让大陆能访问

现状：部署在 Vercel，**大陆被墙**、需翻墙。要稳定访问，主路只有一条：

1. **域名做 ICP 备案（个人）**，用阿里云 / 腾讯云
2. 本站是**纯静态**，可 `next build` 导出后丢到**国内对象存储（OSS/COS）+ CDN**，国内访问飞快、每月几块钱
3. 用 **GitHub Actions** 自动「push → 构建 → 上传国内 + 刷新 CDN」，保持现有写作流不变

- 门槛：备案要实名、约等 1–2 周。
- 不想备案的过渡方案：**香港轻量服务器**（免备案，但慢、不保证稳）。
- ⚠️ 注意：迁国内做静态导出（`output: export`）时，`/keystatic`、`/api/keystatic` 这类动态路由要排除（它们本就只在本地用）；Keystatic 编辑继续在本机跑即可。
- 也可「双线」：海外保留 Vercel，国内加一份备案镜像，用地域 DNS 分流（最全但最复杂）。

## 八、日常运维

- **备份**：代码 + 内容全在 GitHub，自带版本历史，无需额外备份。误删可用 `git` 找回。
- **依赖更新**：`pnpm outdated` 查看；`pnpm up` 升小版本；**大版本（尤其 Next.js）务必谨慎**，升完一定 `pnpm build` 验证再推。
- **监控**：Vercel 后台看部署状态；可在 Vercel 开启 Analytics。
- **排错**：本地 `pnpm build` 能复现 Vercel 的构建错误；构建失败时 Vercel 会保留上一个正常版本，站点不会挂。
- **本地配置**：`.claude/settings.local.json` 已 gitignore（机器相关，不入库）。

## 九、注意事项

- 装依赖只用 **pnpm**，别用 npm（会破坏 `pnpm-lock.yaml`，且本机 npm 在装大依赖时会崩）。
- `/keystatic` 后台**只在本地能用**（线上 404），所以编辑必须在本机进行。
- 日期写成带引号的 `"2026-05-20"`（Keystatic 自动处理；手写 frontmatter 时注意）。
- 文章 URL（slug）用英文短横线，标题可中文。
